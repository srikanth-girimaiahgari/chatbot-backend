DigiMaya Guide

From idea to live operations

DigiMaya is your AI assistant for business DMs. This guide explains, in simple language, what has been built so far, what is already live today, and what DigiMaya is growing into next.

Product: DigiMaya Focus: Business DMs Status: Live and operational

The simple explanation

DigiMaya helps businesses reply to Instagram business messages using their own products, FAQs, and business rules. One system can now support multiple businesses while keeping each business separate.

1. What DigiMaya is right now

Live Today
  • Multi-business Instagram DM auto-reply system
  • Tenant-specific product and FAQ replies
  • Internal provider dashboard for monitoring all businesses
  • Google Sheets sync for reporting
  • Client portal with signup, login, and onboarding
What It Means

DigiMaya is no longer just a single chatbot. It is now a service platform that can support many businesses through one system while giving each business its own setup and visibility.

Current Direction

DigiMaya has already moved into its first digital-employee layer with structured order intents, draft orders, and multi-item order lines. The next layer is payment and order confirmation, but that is being held until the business ownership model is fully clear.

2. How DigiMaya started

  1. DigiMaya first began as a one-business Instagram DM assistant.
  2. It could reply using that business’s products and FAQs.
  3. That proved the main idea worked.
  4. Then it was upgraded so one backend could support many businesses at the same time.

3. The major system we built from scratch

Phase 1

One-business DM assistant

Incoming Instagram messages were received through Meta webhooks, then replied to using stored products and FAQs.

Phase 2

Multi-business foundation

A tenant system was added so each business could have its own Instagram connection, catalog, FAQs, and message flow.

Phase 3

Smarter MAYA replies

MAYA was improved so replies are more product-aware, FAQ-aware, and sales-aware, instead of sounding generic.

Phase 4

Operational control layer

A provider dashboard and reporting sync were added so the service can be monitored and managed properly.

4. What is operational right now

  • Instagram DM routing is live for more than one business.
  • Each business gets replies from its own products and FAQs.
  • You have a provider dashboard to monitor all businesses.
  • You have a Google Sheets sync for simple reporting and review.
  • Clients can access a client portal to sign up, log in, and begin onboarding.
  • Purchase-ready conversations can now become structured order intents.
  • DigiMaya can now create draft orders with multi-item order lines.

5. How DigiMaya works today

  1. A customer sends a message on Instagram.
  2. Meta sends that message to DigiMaya through the webhook.
  3. DigiMaya checks which business owns that Instagram account.
  4. DigiMaya loads that business’s products and FAQs.
  5. MAYA replies using that business’s own information.
  6. If the conversation becomes purchase-ready, DigiMaya creates an order intent and a draft order.
  7. If more than one product is involved, DigiMaya now stores separate order items instead of collapsing them into one fake quantity line.
  8. You can review all activity in the provider dashboard.

6. What the provider side does

The provider side is the control room for DigiMaya. It helps you run the service across all businesses.

  • See all businesses in one place
  • Check setup health
  • Monitor message and handoff activity
  • Review product and FAQ readiness
  • Track order intents and draft orders
  • Use Google Sheets as a simple operating dashboard

7. What the client side now does

  • Create an account
  • Log in
  • Enter business details
  • Set response hours
  • Add products
  • Add FAQs
  • Request help connecting Instagram

This means onboarding is moving from manual setup into a real software flow.
